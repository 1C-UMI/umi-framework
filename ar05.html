Модели и база данных

Подключение к БД и создание структуры таблиц

Так как мы будем хранить информацию о постах блога в базе данных, нам понадобится настроить подключение к базе данных и создать соответствующую таблицу.

Для того чтобы в приложении использовать соединение с базой данных, необходимо подключить ответственный за работу с БД Toolbox и сконфигурировать настройки подключения.

Регистрация используемых в приложении Toolbox состоит в подключении их конфигов в файле /configuration/bootstrap.php. Для работы с БД подключаем компонент DBAL, а так как DBAL использует события, то подключаем и компонент событий: 

/**
* Конфигурация для регистрации наборов инструментов.
*/
Bootstrap::OPTION_TOOLKIT => [
	// ...
	// подключение компонента DBAL
    require($libraryPath . '/dbal/toolbox/config.php'),
    // подключение компонента событий
    require($libraryPath . '/event/toolbox/config.php'),
    // ...
]

В настройках DBAL указывается список используемых в приложении серверов, типы их драйверов БД и настройки подключения. Настройки прописываются в конфигурации приложения в файле /configuration/general.php в опции toolkit:

/**
* Конфигурация для регистрации наборов инструментов.
*/
'toolkit' => [
	// ...
	// конфигурация серверов баз данных
	DbalTools::NAME => [
        'servers' => [
            [
                'id' => 'yourServerId',
                'type' => 'master',
                'driver' => [
                    'type' => 'mysql',
                     'options' => [
                        'dsn'      => 'mysql:dbname=yourDBName;host=yourDBHost',
                        'user'     => 'yourDBUserName',
                        'password' => 'yourDBUserPassword'
                    ]
                ]
            ]
        ],
    ],
    // ...
]

В данном случае используется MySql сервер, база данных 'yourDBName' с пользователем 'yourDBUserName' и паролем 'yourDBUserPassword'.

В указанной в настройках DBAL базе данных создадим таблицу для хранения необходимой информации о посте:

CREATE TABLE `post` (
    `id` bigint(20) unsigned NOT NULL AUTO_INCREMENT,
    `guid` varchar(255) NOT NULL,
    `version` int(10) unsigned DEFAULT '1',
    `type` varchar(255) NOT NULL,
    `content` text,
    `title` varchar(255) DEFAULT NULL,
    `publish_time` int(10) unsigned DEFAULT NULL,
    PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

Чтобы заполнить созданную таблицу демонстрационными данными, добавим в нее три поста:

INSERT INTO `post` VALUES (1, 'c7b42aa9-0e5d-461e-988b-1ea25232145d', 1, 'posts.base', 'First post content', 'First post title', 1386143035);
INSERT INTO `post` VALUES (2, 'dd214a86-df1e-4d8b-888d-c649f71d3dfc', 1, 'posts.base', 'Second post content', 'Second post title', 1386243035);
INSERT INTO `post` VALUES (3, '1a907585-76e7-42a5-9ef3-3c46828d9b0e', 1, 'posts.base', 'Third post content', 'Third post title', 1386253035);

Подключение и настройка ORM

Регистрируем Toolbox ORM в файле /configuration/bootstrap.php: 

/**
* Конфигурация для регистрации наборов инструментов.
*/
Bootstrap::OPTION_TOOLKIT => [
	// ...
	// подключение компонента ORM
    require($libraryPath . '/orm/toolbox/config.php'), 
    // ...
]

В UMI.Framework есть понятие коллекции объектов. Коллекции позволяют совершать операции над принадлежащими им объектами. Для нашего приложения мы будем использовать коллекцию постов блога. Для этого нам нужно зарегестрировать ее в списке коллекций ORM как простую коллекцию posts и описать ее метаданные. Добавляем настройки ORM в конфигурации приложения в файле /configuration/general.php в опции toolkit:

'toolkit' => [
	// ...
	ORMTools::NAME => [
	    'collections' => [
	    	'posts' => [
	    		'type' => ICollectionFactory::TYPE_SIMPLE
	    	]
		],
		'metadata' => [
	    	'posts' => [
				'dataSource' => [
					'sourceName' => 'post'
				],
				'fields' => [
					IObject::FIELD_IDENTIFY => ['type' => IField::TYPE_IDENTIFY, 'columnName' => 'id', 'accessor' => 'getId'],
					IObject::FIELD_GUID => ['type' => IField::TYPE_GUID, 'columnName' => 'guid', 'accessor' => 'getGuid', 'mutator' => 'setGuid'],
					IObject::FIELD_TYPE => ['type' => IField::TYPE_STRING, 'columnName' => 'type', 'accessor' => 'getType', 'readOnly' => true],
					IObject::FIELD_VERSION => ['type' => IField::TYPE_VERSION, 'columnName' => 'version', 'accessor' => 'getVersion', 'mutator' => 'setVersion', 'defaultValue' => 1],
			
					'publishTime' => ['type' => IField::TYPE_INTEGER, 'columnName' => 'publish_time'],
					'title' => ['type' => IField::TYPE_STRING, 'columnName' => 'title'],
					'content' => ['type' => IField::TYPE_TEXT, 'columnName' => 'content']
				],
				'types' => [
					'base' => [
						'fields' => [
							IObject::FIELD_IDENTIFY,
							IObject::FIELD_GUID,
							IObject::FIELD_TYPE,
							IObject::FIELD_VERSION,
							'publishTime', 'title', 'content'
						]
					]
				]
			]
		]
	],
	// ...
]

Создание модели

Реализацию бизнес логики приложения принято помещать в модели. UMI.Framework не накладывает никаких ограничений на название ваших моделей, но, если вы хотите использовать автоматическое внедрение моделей в конструкторы ваших контроллеров, то модель должна реализовывать интерфейс umi\hmvc\model\IModel.

Вся бизнес логика работы демонстрационного приложения заключается в работе с постами блога. Таким образом необходимо создать модель работы с блогом. Модели принято хранить в директории /application/components/имя_компонента/model. Создадим модель posts в файле /application/components/blog/model/PostModel.php:

 
<?php
namespace application\components\blog\model;

use umi\hmvc\model\IModel;
use umi\orm\collection\ICollectionManagerAware;
use umi\orm\collection\TCollectionManagerAware;
use umi\orm\object\IObject;
use umi\orm\selector\ISelector;

/**
 * Модель для работы с постами блога.
 */
class PostModel implements IModel, ICollectionManagerAware {

	use TCollectionManagerAware;

	/**
	 * Получает список всех постов блога.
	 * @return ISelector
	 */
	public function getPosts() {
		return $this->getCollectionManager()->getCollection('posts')
			->select()
				->orderBy('publishTime', ISelector::ORDER_DESC);
	}

	/**
	 * Получает пост блога по его идентификатору.
	 * @param int $guid идентификатор поста
	 * @return IObject пост
	 */
	public function getPost($guid) {
		return $this->getCollectionManager()->getCollection('posts')->get($guid);
	}

	/**
	 * Помечает пост на удаление
	 * @param $postGuid идентификатор поста
	 * @return void
	 */
	public function deletePost($postGuid) {
		$posts = $this->getCollectionManager()->getCollection('posts');
		$posts->delete($posts->get($postGuid));
	}

	/**
	 * Создает новый пост.
	 * @return IObject
	 */
	public function addPost() {
		$post = $this->getCollectionManager()->getCollection('posts')->add();
		$post->setValue('publishTime', time());

		return $post;
	}
}

Созданная модель имеет методы получения, добавления и удаления постов блога.

Регистрация модели в компоненте

Регистрация используемых в компоненте моделей находится в опции [IComponent::OPTION_MODELS]. Для того чтобы зарегестрировать модель постов в компоненте блога, добавим в конфигурацию (/application/components/blog/component.config.php) следущие строки:

// зарегистрированные модели
IComponent::OPTION_MODELS => [
    'post' => __NAMESPACE__ . '\model\PostModel'
],
