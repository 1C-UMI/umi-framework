Компоненты

Структура компонента

Все создаваемые компоненты приложения принято хранить в директории /application/components. Для каждого компонента в директории /application/components следует создать директорию с именем компонента.

Контроллеры, шаблоны, модели и конфигурации форм компонента принято хранить в поддиректориях /controller, /view, /model, /form соответственно.

Создадим следующие директории для компонента блога:

  • /application/components/blog

  • /application/components/blog/controller

  • /application/components/blog/form

  • /application/components/blog/model

  • /application/components/blog/view

Конфигурация компонента

Конфигурации компонентов будем хранить в файлах component.config.php соответствующих директории. Конфигурация описывает такие настройки компонента как используемые контроллеры и модели, дочерние компоненты, маршрутизация и шаблонизация.

Создадим файл /application/components/blog/component.config.php для конфигурации компонента блога, который будем постепенно заполнять:

<?php
namespace application\components\blog;

use umi\hmvc\component\IComponent;
use umi\route\IRouteFactory;

return [

];

Регистрация компонента

Для того чтобы использовать созданный компонент, необходимо зарегестрировать его конфигурацию в списке дочерних компонентов главного компонента (/application/component.config.php) в опции [IComponent::OPTION_COMPONENTS] :

IComponent::OPTION_COMPONENTS => [
    'blog' => '{#lazy:~/application/components/blog/component.config.php}',
]

Конфигурацию компонента блога мы будем наполнять по мере необходимости.
