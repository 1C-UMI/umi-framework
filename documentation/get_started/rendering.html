Отображение

Сетка приложения

Skeleton приложение имеет стандартный контроллер для отображения сетки приложения - application\controller\LayoutController. Для каждого возвращаемого от компонента контроллера ответа, требующего шаблонизации, приложение применяет контроллер сетки.

Порядок формирования ответа пользователю с использованием шаблонизации следующий:

  1. Вызов контроллера компонента

  2. Обработка шаблона, определенного контроллером

  3. Вызов контроллера сетки приложения

  4. Обработка шаблона отображения сетки с использованием результата обработки шаблона компонента

Шаблон сетки приложения находится в файле /application/view/layout.phtml. Место хранения шаблона, тип щаблонизатора и расширения файлов шаблонов было определено в конфигурации основного компонента в секции IComponent::OPTION_VIEW. Конкретный файл был указан при формировании ответа контроллера сетки приложения.

Для шаблонов приложения мы используем фронт-энд фреймворк Foundation версии 4. Этот фреймворк быстро и просто позволит нам создать адаптивную верстку.

Давайте отредактируем сетку приложения, чтобы добавить меню со ссылками на главную страницу и страницу вывода списка постов блога. Для этого в элемент навигации добавим секцию со ссылками и элемент для скрытия меню на мобильных устройствах:

        ->prependFile('/css/normalize.min.css');

<body>

Обратите внимание, что для формирование адресов ссылок мы используем помощник вида (View Helper), генерирующий URL из сгонфигурированных нами маршрутов. Для формирования текста ссылки мы используем помощник шаблона, позволяющий выбирать необходимый текст в зависимости от используемого языка.

Создание отображения для контроллеров блога

Для компонента блога мы будем использовать также php-шаблонизатор, шаблоны будем хранить в директории /application/components/blog/view, файлы шаблонов будут иметь расширение phtml. Укажем эти настройки в конфигурации компонента блога (/application/components/blog/component.config.php), добавив секцию IComponent::OPTION_VIEW:

//...,            
// настройки шаблонизатора
IComponent::OPTION_VIEW        => [
    // тип шаблонизатора
    'type'      => 'php',
    // расширение файлов с шаблонами
    'extension' => 'phtml',
    // путь до шаблонов компонента
    'directory' => __DIR__ . '/view',
],
//...

Пока мы не добавили сами файлы шаблонов, при заходе на страничку со списком постов, мы будем видеть ошибку о том, что обработка шаблона невозможна, так как файл шаблона не доступен на чтение.

Для правильной работы контроллеров нам необходимы следующие шаблоны:

Table 1. Шаблоны компонента blog

ШаблонаОписание
/application/components/blog/view/index.phtmlОтображает список постов
/application/components/blog/view/add.phtmlОтображает форму добавления поста
/application/components/blog/view/edit.phtmlОтображает форму редактирования поста
/application/components/blog/view/delete.phtmlОтображает форму подтверждения удаления поста

Создадим пустые файлы шаблонов, мы будем заполнять их по мере необходимости.

Список постов

Теперь при заходе на страничку списка постов блога, мы видим страницу сетку без контента. Изменим шаблон /application/components/blog/view/index.phtml на следующий, чтобы увидеть список постов блога:

<div class="large-12 columns">
    <h1><?= $this->translate('Blog') ?></h1>
    <div class="row">
        <div class="small-12 columns">
            <a href="<?= $this->url('postAdd', [], true) ?>"><?= $this->translate('Add post') ?></a>
        </div>
    </div>
    <? if ($posts->getTotal()) : ?>
        <? foreach ($posts as $post) : ?>

            <div class="row">
                <div class="small-12 columns">
                    <hr/>
                    <small>
                        <?= date('d.m.Y | H:i', $post->publishTime) ?>
                    </small>
                    <h4>
                        <?= $post->title ?>
                    </h4>
                    <p>
                        <?= $post->content ?>
                    </p>
                    <div class="row">
                        <div class="small-12 columns">
                            <a href="<?= $this->url('postEdit', ['guid' => $post->guid], true) ?>"><small><?= $this->translate('Edit post') ?></small></a>
                            <small>|</small>
                            <a href="<?= $this->url('postDelete', ['guid' => $post->guid], true) ?>"><small><?= $this->translate('Delete post') ?></small></a>
                        </div>
                    </div>
                </div>
             </div>

        <? endforeach; ?>

    <? else : ?>
        <div class="row">
            <div class="small-12 columns">
                <p><?= $this->translate('No posts found') ?></p>
            </div>
        </div>
    <? endif; ?>
</div>

Для того чтобы вывести информацию о постах блога, в шаблоне мы итерируем список постов, который контроллер поместил в переменную $posts. Для генерации ссылок добавления, удаления и редактирования мы опять используем помощник вида для составления URL на основе заданных маршрутов.
