Создание первого контроллера

Теперь, после создания модели, которую мы можем получить в контроллере в качестве сервиса, мы можем приступить к редактированию кода самого простого контроллера - контроллера вывода списка постов.

Контроллер вывода списка постов

В первую очередь дополняем наш контроллер application\components\blog\controller\IndexController конструктором, в котором будем принимать модель постов и свойством, в котором будем хранить модель постов.

/**
 * @var PostModel $postModel модель постов
 */
protected $postModel;

/**
 * Конструктор.
 * @param PostModel $postModel модель постов
 */
public function __construct(PostModel $postModel)
{
    $this->postModel = $postModel;
}

Теперь мы можем использовать нашу модель в основном методе контроллера __invoke(), через который контроллер, собственно, и вызывается.

Цель, которую мы пытаемся достичь, это передать список постов блога с помощью контроллера в представление. Все, что нам нужно сделать для этого - вернуть результат, который может быть обработан шаблонизатором. Для этого модифицируем код метода __invoke() следующим образом:

/**
 * Вызывает контроллер. Передает в контроллер HTTP запрос.
 * @param IComponentRequest $request HTTP запрос
 * @return IComponentResponse результат работы контроллера
 */
public function __invoke(IComponentRequest $request)
{
    return $this->createDisplayResponse(
        'index',
        [
            'posts' => $this->postModel->getPosts()
        ]
    );
}

С помощью метода createDisplayResponse() мы создали ответ компонента с содержимым, требующим отображения, указав имя шаблона index и передав массив переменных, которые можно будет использовать в шаблоне.

Если ответ контроллера не должен проходить процесс шаблонизации, можно формировать результат с использованием метода createPlainResponse(). Это может быть полезным, например в REST-приложениях. Есть возможность выполнить переадресацию с помощью метода createRedirectResponse().

Методы createDisplayResponse(), createPlainResponse() и createRedirectResponse() реализованы в базовом классе контроллера umi\hmvc\controller\type\BaseController.

Общий код контроллера выглядит следующим образом:

<?php
namespace application\components\blog\controller;

use application\components\blog\model\PostModel;
use umi\hmvc\component\request\IComponentRequest;
use umi\hmvc\controller\type\BaseController;

/**
 * Контроллер отображения списка постов блога.
 */
class IndexController extends BaseController
{
    /**
     * @var PostModel $postModel модель постов
     */
    protected $postModel;

    /**
     * Конструктор.
     * @param PostModel $postModel модель постов
     */
    public function __construct(PostModel $postModel)
    {
        $this->postModel = $postModel;
    }

    /**
     * Вызывает контроллер. Передает в контроллер HTTP запрос.
     * @param IComponentRequest $request HTTP запрос
     * @return IComponentResponse результат работы контроллера
     */
    public function __invoke(IComponentRequest $request)
    {
        return $this->createDisplayResponse(
            'index',
            [
                'posts' => $this->postModel->getPosts()
            ]
        );
    }
}

Все, чего нам теперь не хватает, чтобы наконец увидеть результат своей работы, это настройки шаблонизации и сами шаблоны.
