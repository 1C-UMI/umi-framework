Локализация

На данный момент наше приложение при выводе контента на русском языке использует мешанину из русских и английских слов, нам необходимо это исправить. К тому же мы можем перевести и сами посты блога. За перевод лейблов и контента отвечает компонент umi\i18n. Этот компонент уже подключен.

Перевод лейблов

Перевод лейблов осуществляется при формировании отображения на основе шаблонов, если для вывода этих лейблов был использован помощник шаблона translate.

Компонент umi\i18n сконфигурирован в загрузчике приложения так, чтобы он использовал перевод лейблов, находящийся в файле /configuration/i18n.php. Дополним список уже существующих перводов теми, которые мы использовали в процессе разработки приложения:

<?php
return [
    'application' => [
        'ru-RU' => [
            //...,
            'Blog' => 'Блог',
            'Menu' => 'Меню',
            'Add post' => 'Добавить пост',
            'Edit post' => 'Редактировать пост',
            'Delete post' => 'Удалить пост',
            'Save post' => 'Сохранить пост',
            'Title' => 'Заголовок',
            'Content' => 'Содержание',
            'Invalid element value' => 'Некорректное значение элемента',
            'Do you want to delete post?' => 'Вы хотите удалить пост?'
        ],
    ]
];

Переводить можно и лейблы с параметрами, передавая вместе с лейблом список плейсхолдеров. Например, если в вашем шаблоне используется следующая конструкция:

<?= $this->translate('Do you want to delete post "{post}"?', ['post' => '$post->title]) ?>

перевод лейбла может выглядеть так:

'ru-RU' => [
    //...,
    'Do you want to delete post "{post}"?' => 'Вы хотите удалить пост "{post}"?'
]

Перевод контента

В UMI.Framework есть возможность хранить переводы текстовых свойств объектов, чтобы в зависимости от выбранной локали, отображался соответсвующий контент.

Добавим в таблицу post два дополнительных столбца для хранения свойств на русском языке. Для этого выполним следующий запрос:

ALTER TABLE `post` ADD `content_ru` text, ADD `title_ru` varchar(255);

Теперь необходимо изменить метаданные коллекции posts для того, чтобы значения заголовка и контента поста для русской версии брались из колонок title_ru и content_ru. Для этого изменим описание свойств title и content в файле /configuration/general.php:

'title' => [
    'type' => IField::TYPE_STRING,
    'columnName' => 'title',
    'localizations' => [
        'ru-RU' => ['columnName' => 'title_ru'],
        'en-US' => ['columnName' => 'title']
    ],
],
'content' => [
    'type' => IField::TYPE_TEXT,
    'columnName' => 'content',
    'localizations' => [
        'ru-RU' => ['columnName' => 'content_ru'],
        'en-US' => ['columnName' => 'content']
    ],
]

Так как английский язык является языком приложения по умолчанию, то при отсутсвии значений свойств на русском языке, будут выведены значения свойств на английском языке. Поэтому при выводе списка постов блога на русском языке мы до сих пор видим английский контент. Но попробуйте отредактировать любой пост при выбранном русском языке. Изменения будут сохранены в соответствующие колонки русского языка, теперь контент будет разным для английской и русккой версий вашего приложения.

При добавление нового поста в блог запись в базу будет выполнена в соответствующие языку колонки.
