umi\http

Введение

Компонент umi\http предоставляет API для работы с HTTP запросами и ответами сервера.

“HTTP (англ. HyperText Transfer Protocol — «протокол передачи гипертекста») — протокол прикладного уровня передачи данных (изначально — в виде гипертекстовых документов). Основой HTTP является технология «клиент-сервер», то есть предполагается существование потребителей (клиентов), которые инициируют соединение и посылают запрос, и поставщиков (серверов), которые ожидают соединения для получения запроса, производят необходимые действия и возвращают обратно сообщение с результатом.”

HTTP запрос

HTTP запрос - запрос генерируемый клиентом приложения и отправляемый на сервер. Для работы с HTTP запросом в umi\http существует компонент umi\http\request\IRequest. Он предоставляет API, необходимый для корректной обработки запроса клиента.

HTTP запрос содержит в себе адрес ресурса, метод доступа к ресурсу, список параметров и заголовочную информацию.

HTTP ответ

В результате обработки запроса клиента, приложение должно вернуть HTTP ответ. Для работы с HTTP ответом в umi\http существует компонент umi\http\IResponse. Он предоставляет API для работы с HTTP ответом сервера.

HTTP ответ содержит в себе заголовочную информацию (в том числе cookies) и содержимое ответа.

HTTP запрос

Компонент umi\http\request\IRequest предоставляет API для работы с HTTP запросом. Он позволяет оперировать с параметрами и заголовками запроса (запрашивать, устанавливать и т.п.), получить метод запроса и т.п.

Методы запроса

“Метод HTTP (англ. HTTP Method) — последовательность из любых символов, кроме управляющих и разделителей, указывающая на основную операцию над ресурсом. Обычно метод представляет собой короткое английское слово, записанное заглавными буквами. Обратите внимание, что название метода чувствительно к регистру.”

umi\http\request\IRequest::METHOD_CLI

Метод запроса, возвращаемый в случае консольного запуска приложения.

umi\http\request\IRequest::METHOD_GET

Метод запроса - GET. См. также HTTP

umi\http\request\IRequest::METHOD_POST

Метод запроса - POST. См. также HTTP

umi\http\request\IRequest::METHOD_PUT

Метод запроса - PUT. См. также HTTP

umi\http\request\IRequest::METHOD_DELETE

Метод запроса - DELETE. См. также HTTP

umi\http\request\IRequest::METHOD_HEAD

Метод запроса - HEAD. См. также HTTP

Параметры запроса

Одной из самых важных частей запроса являются параметры. Параметры запроса в компоненте umi\http\request\IRequest разделены на контейнеры. Существует несколько типов контейнеров параметров:

umi\http\request\IRequest::GET

GET-параметры запроса. Передаются в адресной строке браузера в виде: /?param1=value&param2=value.

umi\http\request\IRequest::POST

POST-параметры запроса. Передаются в теле запроса, не могут быть изменены пользователем напрямую.

umi\http\request\IRequest::HEADERS

Заголовки запроса. Генерируются браузером клиента.

umi\http\request\IRequest::COOKIE

Cookie запроса. Генерируются приложением и сохраняются у пользователя на определенное время. Не изменяются пользователем напрямую.

umi\http\request\IRequest::FILES

Файлы, отправленные в запросе.

umi\http\request\IRequest::ROUTE

Параметры маршрутизации. Этот контейнер заполняется вручную.

Tip

В случае использования umi\mvc параметры маршрутизации заполняются MVC компонентом, который обрабатывает запрос.

Контейнер, содержащий какие-либо параметры запроса, можно получить используя метод umi\http\request\IRequest::getParams($container). К примеру:

$request->getParams(IRequest::GET);

Не всегда получение контейнера является целесообразным. Например, для получения одного или двух значений параметров можно воспользоваться методом umi\http\request\IRequest::getVar($container, $name, $default = null). К примеру:

$request->getVar(IRequest::GET, 'param1');

HTTP ответ

umi\http\IResponse предоставляет API для работы с HTTP ответом сервера. Он позволяет установить содержимое ответа, изменить отправляемые cookie, установить заголовки и код ответа.

Код ответа

По умолчанию, код ответа сервера имеет значение 200, что означает успешный ответ сервера. Для изменения кода ответа сервера существует метод umi\http\IResponse::setCode($code).

Заголовки

Заголовки ответа сервера могут быть изменены с помощью метода umi\http\IResponse::setHeader($name, $value).

Cookie

Cookie, отправляемые сервером могут быть изменены с помощью метода umi\http\IResponse::setCookie($name, $value, $options). Существует список опций установки cookie:

expire

Время, когда срок действия cookie истекает.

path

Путь к директории на сервере, из которой будут доступны cookie.

domain

Домен, которому доступны cookie. Задание домена 'www.example.com' сделает cookie доступными в поддомене www и поддоменах более высоких порядков

secure

Указывает на то, что значение cookie должно передаваться от клиента по защищенному HTTPS соединению.

httponly

Если задано TRUE, cookie будут доступны только через HTTP протокол. То есть cookie в этом случае не будут доступны скриптовым языкам, вроде JavaScript. Эта возможность была предложена в качестве меры, эффективно снижающей количество краж личных данных посредством XSS атак (несмотря на то, что поддерживается не всеми броузерами). Стоит, однако же, отметить, что вокруг этой возможности часто возникают споры о ее эффективности и целесообразности.

Отправка запроса

Для отправки всех данных запроса, необходимо вызвать метод umi\http\IResponse::send(). Этот метод отправит необходимые заголовки на сервер, а также выведет содержимое.
