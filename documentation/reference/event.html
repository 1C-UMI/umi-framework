umi\event

Введение

Компонент umi\event предназначен для работы с событиями, он позволяет инициировать события, а также назначать и откреплять для этих событий обработчики.

События

События по сути являются системными сообщениями и предназначены для оповещения о том, что что-то произошло или система вошла в некоторое состояние. Каждое событие может быть перехвачено и обработано назначенными для него обработчиками.

События в umi\event представлены интерфейсом umi\event\IEvent.

Каждое событие содержит информацию о том, какой объект был инициатором события (т.е. о том, где событие произошло), тип, однозначно определяющий, какое событие произошло, а также может иметь параметры и теги.

Тип события необходим для того, чтобы назначать обработчики для определенных событий и, когда событие произошло, определять, какие обработчики должны быть вызваны. Теги события могут дополнить его тип. Когда происходит событие, содержащее некоторые теги, для него будут вызваны зарегистрированные для его типа обработчики, теги которых полностью содержатся в тегах события. Если обработчик был зарегистрирован без тегов, то он будет вызван для событий с любыми тегами.

В параметрах события может быть передана дополнительная информация об окружении, где произошло событие, или данные для обработки обработчиком события.

Менеджер событий

Ключевым сервисом компонента umi\event является менеджер событий - umi\event\IEventManager. Каждый класс, который хочет инициировать или обрабатывать события, должен иметь свой менеджер событий. Именно менеджер событий имеет методы для инициализации событий, регистрации обработчиков и вызова зарегистрированных обработчиков.

Обработчики событий

Обработчик события представляет собой callable-функцию, которая описывает все, что должно случиться, когда это событие произойдет. При вызове обработчика ему передается произошедшее событие umi\event\IEvent, поэтому во время обработки события, обработчик может получить информацию о том, где событие произошло и с какими параметрами.

Менеджер событий может формировать цепочки обработчиков, присоединяя к себе другие менеджеры событий. После вызова своих зарегистрированных обработчиков событий вызываются обработчики событий, зарегистрированные в присоединенных менеджерах.

Каждый обработчик цепочки может прервать цепочку, чтобы остальные обработчики событий не были вызваны.

Использование

В качестве примера использования событий рассмотрим обновление информации в заказе интернет-магазина. Задача состоит в том, чтобы в любой момент можно было получить актуальную информацию о количестве товаров в заказе и их общую стоимость, при этом не пересчитывая эти показатели каждый раз. Для этого нужно, чтобы при изменении количества товара в позиции, принадлежащей заказу, заказ узнавал об этих изменениях и обновлял свою информацию.

Классы, которые иницируют или обрабатывают события, должны реализовывать интерфейс umi\event\IEventObservant. Реализация интерфейса umi\event\IEventObservant содержится в трейте umi\event\TEventObservant.

Позиция заказа при изменении количества товара (метод setQuantity()) инициирует событие OrderItemUpdateQuantity, передавая в параметрах событие произошедшее изменение количества.

Код класса позиции заказа:

use umi\event\IEventObservant;
use umi\event\TEventObservant;

/**
 * Позиция заказа
 */
class OrderItem implements IEventObservant
{
    use TEventObservant;

    /**
     * @var string $name наименование товара
     */
    private $name;
    /**
     * @var float $price цена товара
     */
    private $price;
    /**
     * @var int $quantity количество товара
     */
    private $quantity = 0;

    /**
     * Конструктор
     * @param $name наименование
     * @param $price цена
     */
    public function __construct($name, $price)
    {
        $this->name = $name;
        $this->price = (float) $price;
    }

    /**
     * Возвращает цену товара
     * @return float
     */
    public function getPrice()
    {
        return $this->price;
    }

    /**
     * Устанавливает количество товара
     * @param int $quantity
     */
    public function setQuantity($quantity)
    {
        $quantity = (int) $quantity;
        if ($this->quantity != $quantity) {

            $this->fireEvent(
                'OrderItemUpdateQuantity',
                [
                    'quantityDelta' => $quantity - $this->quantity
                ]
            );

            $this->quantity = $quantity;
        }
    }
}

Заказ при добавлении позиции заказа (метод appendItem()) подписывается на события этой позиции. Метод bindLocalEvents() описывает, какие события будет "слушать" заказ и какие обработчики будет вызывать. Заказ подписан на событие типа OrderItemUpdateQuantity, когда это событие происходит, заказ обновляет информацию об общем количестве товаров и их стоимости.

Код класса заказа:

use umi\event\IEvent;
use umi\event\IEventObservant;
use umi\event\TEventObservant;

/**
 * Заказ
 */
class Order implements IEventObservant
{
    use TEventObservant;

    /**
     * @var int $quantity количество товаров
     */
    private $totalQuantity = 0;

    /**
     * @var float $totalPrice сумма заказа
     */
    private $totalPrice = 0;

    /**
     * Добавляет позицию в заказ
     * @param OrderItem $item
     */
    public function appendItem(OrderItem $item)
    {
        $this->subscribeTo($item);
    }

    /**
     * Возвращает общее количество товаров в заказе
     * @return int
     */
    public function getTotalQuantity()
    {
        return $this->totalQuantity;
    }

    /**
     * Возвращает суммарную стоимость заказа
     * @return float
     */
    public function getTotalPrice()
    {
        return $this->totalPrice;
    }

    /**
     * Устанавливает локальные обработчики событий
     * @return $this
     */
    protected function bindLocalEvents()
    {
       $this->bindEvent(
           'OrderItemUpdateQuantity',
           function (IEvent $event) {
               /**
                * @var OrderItem $orderItem
                */
               $orderItem = $event->getTarget();
               $quantityDelta = $event->getParam('quantityDelta');

               $this->totalQuantity += $quantityDelta;
               $this->totalPrice += ($orderItem->getPrice() * $quantityDelta);
           }
       );
    }
}

Предположим, мы хотим купить молоко. Создадим заказ и добавим в него молоко по цене 34,5 рубля. Подумаем и скажем, что нам нужно три пакета молока. В итоге общее количество товара составляет 3 штуки, а общая стоимость - 103,5 рублей.

$order = new Order();
$milkItem = new OrderItem('Молоко', '34.5');

$order->appendItem($milkItem);
$milkItem->setQuantity(3);

echo $order->getTotalQuantity() . PHP_EOL; //3
echo $order->getTotalPrice(); //103.5
