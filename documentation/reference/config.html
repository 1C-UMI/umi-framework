umi\config

Введение

Компонент umi\config предоставляет возможности по загрузке и сохранению конфигурационных файлов, предоставляет интерфейс для работы с загруженной конфигурацией.

С помощью компонента umi\config можно организовать структуру хранения конфигурации. Конфигурация разделяется на глобальную(основную, master) и локальную(local). Глобальная конфигурация предоставляется разработчиком, она одинакова на серверах разработки, тестирования и основном сервере, локальная конфигурация уникальна для каждого из серверов, и содержит уникальную для них информацию (например, настройка соединения с БД).

Компонент umi\config позволяет разделить глобальную конфигурацию на части, которые могут загружаться лениво. Каждая из частей конфигурации может быть переопределена в локальной конфигурации.

Important

Формат разбиение конфигурации на файлы является общим для глобальной и локальной конфигурации и задается глобальной конфигурацией.

Конфигурация

Компонент umi\config предоставляет интерфейс umi\config\entity\IConfig для работы с конфигурацией.

umi\config\entity\IConfig хранит значения конфигурации 2х типов: локальное и мастер. Методы модификации позволяют изменять локальное значение по своему усмотрению, значения же мастер конфигурации не могут быть изменены с помощью данного интерфейса. Измененные значения не сохраняются до непосредственной записи в конфигурацию, т.е. не кэшируются.

umi\config\entity\IConfig содержит методы получения, установки, удаления, сброса или проверки значения из конфигурации или части конфигурации по заданному пути. Путь до значения задается строкой из ключей разделенных символом umi\config\entity\IConfig::PATH_SEPARATOR = '/'.

Пример работы с конфигурацией:

$config = $configIO->read('~/config.php');

var_dump($config['key']);

$config['key'] = 'Hello world';
var_dump($config['key']); // string(11) "Hello world"

var_dump($config['subconfig/subkey1']);

Доступ к значениям конфигурации может осуществляться как с помощью методов, так и с помощью \ArrayAccess интерфейса.

Конфигурация может быть преобразована в массив(распакована) с помощью метода toArray, при этом все лениво загружаемые конфигурации будут автоматически загружены.

Warning

Для больших конфигураций не рекомендуется выполнять преобразование в массив, так как это негативно сказывается на производительности и используемой памяти.

I/O сервис конфигурации

Компонент umi\config предоставляет сервис для операций ввода/вывода с конфигурациями. Этот сервис позволяет:

  • Прочитать конфигурацию

  • Записать локальную конфигурацию

Пример работы с сервисом ввода-вывода:

$config = $configIO->read('~/config.php'); // чтение конфигурации

$configIO->write($config); // запись конфигурации

I/O сервис конфигурации работает с символическими именами, которые регистрируются в нем. Каждое символическое имя конфигурации ссылается на имя основной(master) директории и, опционально, на имя локальной директории. Таким образом символическое имя файла может ссылаться сразу на несколько реальных файлов- локальный и основной. Все операции ввода/вывода конфигурации происходят по ее символическому имени.

Tip

Для ясности, рекомендуется начинать символические имена директорий с символа "~".

Пример регистрации символических имен директорий:

$configIO->registerAlias('~', __DIR__ . '/config/master', __DIR__ . '/config/local');
$configIO->registerAlias('~/application', __DIR__ . '/appconfig/master', __DIR__ . '/appconfig/local');

При загрузке конфигурации происходит слияние основного и локального конфигурационных файлов. Запись конфигурации производит сохранение локальных конфигурационных файлов. Соответственно, структура разделения, заданная основными конфигурационными файлами, дублируется при сохранении.

Tip

Локальная директория является необязательной. Соответственно при ее отсутствии никакого слияния происходить не будет.

Кэширование

Сервис I/O для работы с конфигурацией поддерживает кэширование. Для этого полученная конфигурация сериализуется и сохраняется в файл, при повторном чтении операция слияния локальных и мастер конфигураций происходить не будет. Лениво загружаемые конфигурации кэшируются в отдельных файлах, тем самым позволяя избежать лишнего использования памяти при загрузке из кэша.

Настройка кэширования происходит в конфигурации набора инструментов работы с конфигурацией. За настройку кэширования отвечают директивы:

ConfigTools::hasCache

Включает или выключает кэширующий механизм для конфигурации.

ConfigTools::cache

Настройки кэширующего механизма конфигурации.

Пример настроек кэширования для компонента umi\config

<?php
return [
    'hasCache' => true,
    'cache' => [
        'directory' => __DIR__ . '/cache',
    ]
];

Форматы конфигурационных файлов

Все форматы конфигурационных файлов поддерживают разбиение конфигурации на части. Части конфигурации могут быть следующих типов:

umi\config\io\reader\IReader::COMMAND_PART

Часть конфигурации. Загружается автоматически вместе с конфигурацией.

umi\config\io\reader\IReader::COMMAND_LAZY

Лениво загружаемая часть конфигурации.

PHP

Конфигурация в данном формате представляет из себя массив с данными. Простым примером конфигурации в этом формате является файл, со следующим содержимым:

<?php
return [
    'key' => 'value'
];

Для поддержки разделения конфигурации на несколько файлов существуют управляющие конструкции. Формат управляющей конструкции: '{#<type>:<file>}', где <type> - тип подключение конфигурации, а <file> - символическое имя файла подключаемой конфигурации. Пример конфигурации, разделенной на части:

<?php
return [
    'components'  => [
        'component1'     => '{#partial:~/components/component1.config.php}',
        'component2'     => '{#lazy:~/components/component2.config.php}',
        'component3'     => '{#lazy:~/components/component3.config.php}',
    ],
];
