umi\route

Введение

Компонент umi\route предоставляет API для маршрутизации. Маршрутизация - это процесс определения маршрута. Задачами этого компонента являются определение маршрута по заданному пути, либо построение пути по заданному маршруту.

Маршрутизатор

umi\route\IRouter является основной сущностью компонента umi\route. Его непосредственной задачей является разбор и построение маршрутов. Имена маршрутов задаются в виде дерева, состоящего из частей маршрута umi\route\type\IRoute. При поиске пути маршрутизатор производит последовательный обход дерева маршрутов.

В результате разбора маршрутизатор возвращает объект результата разбора - umi\route\result\IRouteResult.

Маршруты

Задание маршрута задается при помощи частей - umi\route\type\IRoute. Задачей части маршрута является проверка пути на соответствие этой части.

Каждый маршрут задается набором опций формата:

[
	'type' => '<type>',
	'route' => '<route>',
	'defaults' => [],
       'subroutes' => []
]

Где <type> - тип маршрута, <route> - маска маршрута, <defaults> - список параметров по умолчанию, <subroutes> - массив конфигураций дочерних маршрутов.

Результат разбора

Результат разбора umi\route\result\IRouteResult содержит в себе информацию о параметрах, полученных в результате разбора - имя маршрута, который был найден, параметры, а также подошедшую и не подошедшую часть пути.

Использование

Создание маршрутизатора

Создание (или настройка) маршрутизатора, происходит путем задания ему дерева, состоящего из частей маршрутов. Такое дерево описывается с помощью конфигурации. Например:

return [
    'aboutUs' => [
        'type'     => IRouteFactory::ROUTE_FIXED,
        'route'    => '/about',
        'defaults' => [
            'controller' => 'aboutUs'
        ]
    ],
    'base'    => [
        'type'     => IRouteFactory::ROUTE_FIXED,
        'defaults' => [
            'controller' => 'index'
        ]
    ],
];

Важно

Порядок указания маршрутов важен! Проверка маршрутов идет в порядке задания маршрутов до первого совпадения.

Маршрутизатор выполняет проверку маршрутов первого уровня, до первого совпадения. Когда совпадение найдено, то маршрутизатор выполняет аналогичные действия с дочерними маршрутами найденного маршрута.

Маршрутизатор, созданный с помощью данной конфигурации, содержит два маршрута base и aboutUs. Создать маршрутизатор на основе конфигурации можно следующим образом:

$router = $routeFactory->createRouter($config);

Разбор маршрута

Маршрутизатор может строить или разбирать маршрут. Рассмотрим маршрутизатор, созданный в предыдущей главе.

Данный маршрутизатор умеет разбирать и строить 2 маршрута - base и aboutUs. Рассмотрим пример разбора маршрута:

$result = $router->match('/about');
var_dump($result);

В результате, на экран будет выведено:

class umi\route\result\RouteResult#42 (4) {
  protected $route => string(7) "aboutUs"
  protected $matches => array(1) {
    'controller' => string(7) "aboutUs"
  }
  protected $matchedUrl => string(6) "/about"
  protected $unmatchedUrl => NULL
}

Составление маршрута

Для составления маршрута, необходимо указать имя маршрута и набор необходимых параметров. Например:

var_dump($router->assemble('aboutUs')); // string(6) "/about"

При составлении маршрута для типов маршрутов с динамическими параметрами, необходимо также задать обязательные параметры. Обязательным называется параметр, который используется при построении и разборе маршрута, но не указан в секции defaults.

Типы маршрутов

Для более гибкого задания маршрута, в компоненте umi\route существует расширяемый набор типов маршрутов. Каждый тип маршрута имеет путь и набор параметров по умолчанию.

Фиксированный маршрут

Фиксированный маршрут имеет тип umi\route\IRouteFactory::ROUTE_FIXED и задает статический маршрут, который не имеет динамически изменяемых параметров. Пример такого маршрута: /fixed/route. При разборе такого типа маршрутов будет возвращен набор параметров, указанный в defaults, без изменений.

Маршрут на основе регулярных выражений

Тип маршрута, использующий для проверки маску, являющуюся регулярным выражением. Данный маршрут имеет динамически изменяемые параметры. Они будут объединены с параметрами по-умолчанию в результате работы маршрутизатора.

Важно

Динамически изменяемые параметры, указанные в маршруте, должны быть именованными. Неименованные совпавшие части регулярного выражения возвращены не будут.

Пример маршрута на основе регулярных выражений: /route/regexp/(?P<id>(\d+)). В данном маршруте динамически задается обязательный параметр id.

Маршрут на основе simple выражений

Simple выражения - формат выражений для задания маршрутов. Он базируется на маршруте, основанном на регулярных выражениях. Т.е. любой маршрут заданного типа может быть представлен в виде регулярного выражения.

В данном формате, каждый параметр маршрута задается в виде: {имя_параметра:тип_параметра}. Тип параметра может быть опущен, в этом случае по умолчанию будет использован тип строка. Доступные типы параметров для simple выражений:

integer

Целое число.

float

Число с плавающей точкой.

string

Строка. В строке не может содержаться символ /.

text

Строка, которая может содержать любые символы.

guid

GUID идентификатор.

Пример маршрута заданного типа: /simple/route/{controller}/{module:string}.

Подсказка

Важной частью маршрутов данного типа является возможность опускать параметры по умолчанию при разборе и создании пути. Таким образом, если в указанном примере параметры controller и module указаны в секции defaults, то собранный маршрут будет выглядеть /simple/route. Этот путь также соответсвует маршруту с данной маской.

Расширенный(extended) маршрут на основе simple выражений

В отличии от простых simple маршрутов, extended маршрут не имеет указания типов параметров внутри маски маршрута. Типы параметров задаются регулярными выражениями в отдельной секции конфигурации маршрута - rules. Таким образом, маршруты extended типа являются более гибкой записью маршрутов simple типа.

Пример маршрута: /{component}/{controller}. Пример конфигурации:

'base' => [
	'type' => 'extended',
	'route' => '/{module}/{controller}',
	'rules' => [
		'module' => '.+?',
		'controller' => '.+?'
	]
]

Подсказка

Для маршрутов extended типа также действуют правила с необязательными аргументами, аналогичные simple

Расширенное использование

Вложенные маршруты

При составлении маршрута, части маршрута могут быть вложены друг в друга. При этом, параметры таких маршрутов складываются, а имя маршрута составляется из имен частей маршрутов, разделенных символом /. Рассмотри пример маршрутизатора со вложенными маршрутами:

return [
    'base' => [
        'type'      => IRouteFactory::ROUTE_FIXED,
        'route'     => '/',
        'defaults'  => [
            'component'  => 'application',
            'controller' => 'index'
        ],
        'subroutes' => [
            'edit' => [
                'type'     => IRouteFactory::ROUTE_FIXED,
                'route'    => 'edit',
                'defaults' => [
                    'controller' => 'edit'
                ]
            ]
        ]
    ],
];

Такая конфигурация описывает маршрут base и дочерний маршрут edit. Заметим, что дочерний маршрут edit переопределяет параметр controller секции defaults. Таким образом, в результате разбора маршрута /edit маршрутизатор вернет следующий результат:

class umi\route\result\RouteResult#39 (4) {
  protected $route => string(9) "base/edit"
  protected $matches => array(2) {
    'controller' => string(4) "edit"
    'component' =>  string(11) "application"
  }
  protected $matchedUrl => string(5) "/edit"
  protected $unmatchedUrl => NULL
}

Дочерняя часть маршрута не является обязательной частью. Таким образом, данный маршрутизатор умеет разбирать маршрут /. В результате разбора этого маршрута, маршрутизатор вернет следующий результат:

class umi\route\result\RouteResult#41 (4) {
  protected $route => string(4) "base"
  protected $matches => array(2) {
    'component' => string(11) "application"
    'controller' => string(5) "index"
  }
  protected $matchedUrl => string(1) "/"
  protected $unmatchedUrl => NULL
}

При создании маршрута, имена частей разделяются символом /. Например:

var_dump($router->assemble('base/edit')); // string(5) "/edit"
