umi\session

Введение

Механизм сессий является важным механизмом работы Web приложения. Он позволят хранить временные данные о пользователе, в процессе посещения им ресурса. Современные web приложения не мыслимы без использования сессий. Подробнее про сессии можно прочитать в оф. документации.

Компонент umi\session предоставляет разработчику API для удобной работы с механизмом сессий. Он реализует ленивый запуск сессии, а также разбивает контейнер значений сессии на пространства имен(umi\session\ISessionNamespace. Для каждого из пространств имен возможно осуществить валидацию (например по времени или по User Agent браузера), для этого существует umi\session\validator\ISessionValidator. Непосредственную работу по запуску, закрытию и т.п. сессии производит менеджер сессии - umi\session\ISessionManager.

Ленивый запуск сессии

Как было сказано выше, сессия запускается ленивым образом, что отлично сказывается на быстродействии системы.

При первом заходе клиента на страницу сессий не существует. В такой ситуации, сессия не будет запущена до того момента, пока в нее не будет записано хотя бы одно значение. Это значит, что при регистрации и создании пространств имен в сессии запуска не происходит. Сессия не будет запущена в этом случае даже при попытке чтения из пространства имен, или записи метаданных в пространство имен (например при установке валидатора).

В последующие посещения страницы, если сессия была запущена, то она будет восстановлена (т.е. автоматически запустится).

Пространства имен сессии

Для структуирования данных контейнера сессии были созданы пространства имен. Они позволяют разделить контейнер на несколько частей, что позитивно сказывается на конфликтах имен и др.

Каждое пространство имен предстваляет из себя совокупность метаданных пространства имен и данных. Метаданные предназначены для системного использования и не гарантируют сохранения данных, записаных в них. Данные пространства имен, напротив, гарантируют сохранение содержимого, до момента инвалидации сессии.

Каждое пространство имен сессии должно быть зарегистрировано, перед использованием. При регистрации указывается имя пространства имен и список его валидаторов.

Валидация данных сессии

Любое пространство имен может иметь набор валидаторов. Валидаторы указываются при регистрации пространства имен. При создании пространства имен оно прогоняется через список своих валидаторов. В случае если хотя бы один валидатор не подходит, пространство имен очищается.

Базовое использование

Важно

При создании компонента umi\session происходит определение необходимости запуска сессии. В случае таковой необходимости сессия будет автоматически запущена.

Пространства имен

Попробуем зарегистрировать пространство имен сесии и записать в него значение:

$sampleNs = $session
    ->registerNamespace('sample')
    ->getNamespace('sample');

var_dump($session->hasNamespace('sample')); // выведет "1"

$sampleNs['key'] = 'value'; //записываем пространство имен
var_dump($sampleNs->toArray());

Менеджер сессии

Менеджер сессии занимается работой с состоянием сессии. Как уже говорилось выше, старт сессии происходит в автоматическом режиме, по этому, в большинстве случаев вам не придется обращаться к менеджеру сессии напрямую. Но в случае, если вам нужно принудительно запустить сессию или уничтожить её вы сможете это сделать используя менеджер.

Рассмотрим пример использования менеджера сессий:

var_dump($sessionManager->start());
var_dump($sessionManager->destroy());
var_dump($sessionManager->destroy());

Расширенное использование

Использование валидаторов

При регистрации пространства имен сессии может быть указан список валидаторов. К примеру:

$session->registerNamespace(
        'sample',
        [
            ISessionValidator::LIFE_TIME => 60
        ]
    );

В результате будет зарегистрировано пространство имен сессии с ограничением времени жизни в 60 секунд. Это означает, что при отсутсвии активности пользователя в течении 60 секунд, пространсво имен будет уничтожено.

Доступные валидаторы

ISessionValidator::TYPE_LIFE_TIME => TIME

Валидатор проверки пространства имен сессии по времени последнего доступа. В случае, если к контейнеру не обращались TIME времени, то данные контейнера сессии будут уничтожены.

ISessionValidator::TYPE_USER_AGENT

Валидатор осуществляет проверку пространства имен сессии по заголовку USER AGENT браузера. Таким образом, валидатор реализует защиту от атаки на пользователя, путем кражи идентификатора сессии.
